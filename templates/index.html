🎓 Query Optimization Educational Guide

                                
This guide explains the foundational relational database optimizer concepts simulated in this application.

                            

                            
                            

                                

                                    
 Access Paths (Scans)

                                    
    
                                        
  • Sequential Scan (Seq Scan): Reads the entire table from beginning to end. Highly efficient for small tables but has linear time complexity O(N) for large tables.
    • 
                                        
  • Index Scan: Uses a B-Tree or Hash Index to jump directly to target records. Simulated here for primary key id. Features logarithmic time complexity O(log N).
    • 
                                    

                                

                                
                                

                                    
 Join Algorithms

                                    
    
                                        
  • Nested Loop Join: For each row in outer table, scans inner table. Cost is O(M * N). Good for small tables or when inner table has indexes.
    • 
                                        
  • Hash Join: Builds an in-memory hash map of the smaller table, then probes it with the larger table. Linear complexity O(M + N). Very efficient for large unsorted sets.
    • 
                                        
  • Sort Merge Join: Sorts both relations by join keys, then merges them in a single pass. Cost is O(M log M + N log N). Excellent when inputs are already sorted.
    • 
                                    

                                

                                
                                

                                    
 Logical Rewrite Rules

                                    
    
                                        
  • Selection Pushdown: Moves WHERE filters down past joins, applying them immediately at table scans. Reduces intermediate relation sizes early, avoiding expensive Cartesian products.
    • 
                                        
  • Projection Pushdown: Restricts column fetching to only referenced columns. Minimizes memory and I/O bus usage.
    • 
                                        
  • Join Reordering: Calculates alternative join orders (e.g. joining small tables first) to reduce the rows fed into subsequent join operators.
    • 
                                    

                                

                                
                                

                                    
 Cost-Based Optimization

                                    
    
                                        
  • DBMS query optimizer evaluates physical plan candidates using statistics (row counts, selectivity).
    • 
                                        
  • Selects the cheapest plan using cost formulas (disk I/O page reads + CPU execution cycles).
    •
